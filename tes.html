Evaluasi Model Klasifikasi untuk Prediksi Kualitas Anggur#

Pendahuluan#

Kualitas anggur merupakan salah satu faktor penting dalam industri makanan dan minuman, khususnya dalam produksi wine. Penilaian kualitas anggur biasanya dilakukan oleh para ahli dengan menggunakan metode sensorik yang bersifat subjektif dan membutuhkan waktu serta biaya yang tidak sedikit. Oleh karena itu, pendekatan berbasis data dan teknologi machine learning menjadi solusi alternatif yang menjanjikan untuk memprediksi kualitas anggur secara otomatis dan objektif.

Dalam penelitian ini, dilakukan evaluasi terhadap beberapa model klasifikasi untuk memprediksi kualitas anggur berdasarkan fitur-fitur kimia yang terdapat dalam dataset Wine Quality. Dataset ini berisi berbagai parameter seperti keasaman, kadar gula, alkohol, dan lainnya yang diperoleh dari sampel anggur merah dan putih. Tujuannya adalah untuk mengetahui sejauh mana model klasifikasi dapat membedakan tingkat kualitas anggur secara akurat.

Melalui evaluasi ini, diharapkan dapat ditemukan model yang paling optimal dalam memprediksi kualitas anggur serta memahami faktor-faktor apa saja yang paling memengaruhi hasil prediksi. Dengan demikian, pendekatan ini dapat memberikan kontribusi positif dalam mendukung pengambilan keputusan di bidang produksi dan pengolahan anggur.

Data Understanding

Dataset

Dataset yang digunakan adalah Wine Quality Dataset yang tersedia di UCI Machine Learning Repository. Dataset ini berisi data hasil uji kimia dari sampel anggur merah dan putih yang telah dinilai kualitasnya oleh para ahli.

Link dataset: https://archive.ics.uci.edu/ml/datasets/Wine+Quality
Jumlah data: 6497 baris data
Jumlah fitur: 12 fitur input + 1 label output (quality)
Dua file utama: winequality-red.csv dan winequality-white.csv

Tujuan

  • Memprediksi kualitas anggur berdasarkan fitur-fitur kimia.
  • Membandingkan performa beberapa model klasifikasi.
  • Mengetahui fitur yang paling memengaruhi kualitas anggur.
  • Menilai akurasi dan performa model menggunakan metrik evaluasi.

Identifikasi Kualitas Dataset#

Deteksi Missing Value#

Missing value merupakan data yang hilang pada suatu dataset. Hal ini bisa terjadi oleh beberapa faktor, diantaranya adalah :

  • Interviewer recording error terjadi akibat kelalaian petugas pengumpul data (pewawancara), misalnya ada sejumlah pertanyaan yang terlewatkan.

  • Respondent inability error terjadi akibat ketidakmampuan responden dalam memberikan jawaban akurat, misalnya karena tidak memahami pertanyaan, bosan atau kelelahan (respondent fatigue) akhirnya responden mengosongkan sejumlah pertanyaan atau berhenti mengisi kuesioner di tengah jalan.

  • Unwillingness respondent error terjadi karena responden tidak berkenan memberikan jawaban yang akurat, misalnya pertanyaan soal penghasilan, usia, berat badan, pengalaman melakukan pelanggaran hukum, dll.

Cara penanganan Missing Values :

  • Mengabaikan dan membuang missing data.

  • Estimasi parameter.

  • Imputasi.

Dalam modul kali ini saya akan menangani missing values menggunakan metode imputasi, yang ada 2 metode yaitu menggunakan metode Mean dan menggunakan metode KNN.

pada kolom di bawah ini akan dilakukan pendeteksian missing values terlebih dahulu pada masing-masing kolom.

Preprocessing Data#

Modeling#

import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import classification_report, confusion_matrix

# Misal nama DataFrame-mu adalah `data`
X = df.drop('quality', axis=1)
y = df['quality']

# Split data
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)

# Scaling (wajib untuk SVM dan Naive Bayes)
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

Decision Tree#

from sklearn.tree import DecisionTreeClassifier

dt_model = DecisionTreeClassifier(random_state=42)

dt_pred = dt_model.predict(X_test)

print("🎯 Decision Tree")
print(confusion_matrix(y_test, dt_pred))
print(classification_report(y_test, dt_pred))
